## 2. Phase 1: Setup & Foundational Tasks

There are no specific setup or foundational tasks required for this feature, as it builds upon the existing project structure. All implementation will occur within the existing `MazeService`.

## 3. Phase 2: Core Implementation (User Story 1)

This phase covers all functional requirements for generating and managing the maze lifecycle.

**User Story Goal**: As a player, when I start a game, I want to be teleported into a fully generated, physical maze structure so that I can begin navigating and solving it.

**Independent Test Criteria**:
- A maze model can be successfully generated by calling `MazeService.generate("Medium")`.
- The generated model contains multiple `Part` instances for walls and a floor.
- The generated model can be successfully removed from the workspace by calling `MazeService.destroy(model)`.

- [x] T001 [US1] Implement the internal grid data structure initialization in `src/ServerScriptService/Services/MazeService.lua`
- [x] T002 [US1] Implement the Recursive Backtracker algorithm for path carving in `src/ServerScriptService/Services/MazeService.lua`
